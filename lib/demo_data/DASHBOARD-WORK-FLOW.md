Các file DASHBOARD.PAGEDATA -> DASHBOARD.LST -> DASHBOARD.DTLS -> DASHBOARD.CHARTDTLS lần lượt là các request mẫu gồm tương ứng các payload và response tương ứng của các request đó để giúp bạn phân tích ý nghĩa và cách áp dụng để xây dựng chức năng DASHBOARD trên phiên bản web mà tôi có.
Phiên bản web hiện tại xử lý MUIX Chart để xây dựng các Chart Dashboard theo các Data tương ứng trong các file trên và tôi muốn chuyển các workflow sang phiên bản mobile flutter tương ứng sử dụng fl_chart.

1. Workflow hiện tại chạy như theo thứ tự sau DASHBOARD.PAGEDATA -> DASHBOARD.LST -> DASHBOARD.DTLS -> DASHBOARD.CHARTDTLS trong đó chi tiết như sau:
    1.1. Đầu tiên vối DASHBOARD.PAGEDATA khi load module này lên cấu trúc tương tự khá giống các module khác trong ứng dụng nhưng riêng Dashboard sẽ khác 1 tí nên có thể sẽ làm riêng biệt so với cấu trúc chung của các module khác: Với payload tương tự xem cấu trúc payload qua file DASHBOARD.PAGEDATA-Payload.json và response tương tự qua DASHBOARD.PAGEDATA-Response.json trong đó quan trọng ở key chartConfigs.data đây là list chart sẽ có tùy thuộc vào user đang log sẽ khác nhau list chart này như select dropdown để chọn loại chart để xem nhưng cấu trúc như tree để lọc và tìm kiếm chart tương ứng để show ra và khi chọn từ dropdown này chart sẽ load lại theo request DASHBOARD.CHARTDTLS tương ứng đó là data được lấy từ dropdown data chartConfigs.data khi chọn
    1.2. DASHBOARD.LST là 1 request dạng Inbox data chạy riêng biệt có thiết kế dạng card data có thể scroll ngang cho dễ nhìn và chạy độc lập trong đó key data trong response chính là data chính của list này và thiết kế từng item là 1 card data inbox show ra. trong đó payload mặc định sẽ lấy theo năm hiện tại (Back end xử lý) nếu không truyền vào key year và sẽ load lại data tương ứng khác khi chọn key year làm dropdown sẽ load lại request DASHBOARD.LST ví dụ year: 2025
    1.3. DASHBOARD.DTLS là danh sách list chart default sẽ show ra mặc định giao diện có thể add thêm chart hoặc remove bớt nếu cần thiết trong đó list chart default sẽ dựa theo key itemDetail.data
    1.4. DASHBOARD.CHARTDTLS là request load chi tiết chart được chọn với key filterValues khi không truyền sẽ lấy default filterValues hoặc sẽ lấy theo tiêu chí filter nếu truyền vào key trên theo value mẫu như sau: filterValues: {
        "@newYear": "2025",
        "@user": "A2C8A98F-25F3-454E-8D9A-E572502F232D"
    }
